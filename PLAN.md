# PLAN.md

본 문서는 챗봇 API 서비스 구축을 위한 단계별 구현 계획을 정리한 문서입니다. 모든 개발은 Hexagonal Architecture 및 TDD 원칙을 준수하며 진행됩니다.

## Phase 1. 프로젝트 기초 설정 및 사용자 인증 (Auth)

- **목표:** 프로젝트 기본 구조를 설정하고, 보안 계층(JWT)과 사용자 관리 기능을 구현합니다.
- **주요 내용:**
    - Gradle 기반 프로젝트 설정 및 라이브러리 의존성 추가.
    - Hexagonal Architecture 구조에 따른 패키지 구성.
    - 사용자(User) 도메인 및 리포지토리 구현 (신규 가입 시 기본 권한 MEMBER).
    - 초기 데이터(`data.sql`)를 통한 테스트용 관리자 계정 설정 및 관련 스프링 부트 프로퍼티 구성.
    - JWT 발급, 검증 및 Spring Security 통합 인증 체계 구축 (Access 5분, Refresh 24시간).
    - 회원가입, 로그인, 토큰 재발급 API 구현.

## Phase 2. 대화 관리 기능 (Chat & Thread)

- **목표:** AI와의 대화 흐름을 관리하고, 스레드 정책에 따른 컨텍스트 유지를 구현합니다.
- **주요 내용:**
    - 스레드(Thread) 및 대화(Chat) 도메인 구현.
    - 마지막 질문 후 30분 유효 시간 체크 로직을 포함한 스레드 자동 생성/유지 엔진 개발.
    - OpenAI API 연동 (비동기 및 WebClient 활용).
    - 스트리밍 응답(SSE) 및 일반 응답 처리 모듈 개발.
    - 페이징 기반 대화 목록 조회 및 스레드 삭제 기능 구현.

## Phase 3. 사용자 피드백 관리 (Feedback)

- **목표:** 생성된 대화에 대한 사용자 만족도를 기록하고 관리자 처리가 가능하도록 구현합니다.
- **주요 내용:**
    - 피드백(Feedback) 도메인 구현 및 비즈니스 규칙(1인 1대화 1피드백) 적용.
    - 피드백 생성, 조회(필터링 포함) API 개발.
    - 관리자 전용 피드백 상태 변경(PATCH) 기능 구현.

## Phase 4. 분석 및 데이터 보고 (Admin & Statistics)

- **목표:** 시스템 운영 현황 파악을 위한 통계 수집 및 데이터 추출 기능을 제공합니다.
- **주요 내용:**
    - 사용자 활동(가입, 로그인, 대화) 로그 기록 및 집계 로직 개발.
    - ISO 8601 기간 기반 통계 조회 API 구현.
    - 대화 목록 CSV 보고서 생성 및 파일 다운로드 스트림 처리.

## Phase 5. 품질 최적화 및 문서화 (Refactor & Docs)

- **목표:** 코드의 완성도를 높이고 최종 시연 및 배포를 위한 준비를 마칩니다.
- **주요 내용:**
    - 통합 테스트 수행 및 성능 병목 지점 개선.
    - REST Docs 또는 Swagger를 활용한 가독성 높은 API 문서 제공.
